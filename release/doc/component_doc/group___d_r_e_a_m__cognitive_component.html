Component Description

The current component is the implementation of the attentionReactionSystemGUI placeholder component.

This component is used to provide with simulated inputs to the attentionReaction component. The GUI has been used to simulate the sensory information of the robot as a first step in the implementation of this system. It should be replace by a sensory information simulator of an specific robot, i.e. Nao robot. But for the purpose of validating the attentionReaction subsystem the GUI is an equally good option.

Parameters

Command-line Parameters

The following key-value pairs can be specified as command-line parameters by prefixing – to the key e.g. –from file.ini. The value part can be changed to suit your needs; the default values are shown below.

  • from attentionReactionSystemGUI.ini
    • specifies the configuration file
  • context components/attentionReactionSystemGUI/config
    • specifies the sub-path from $DREAM_ROOT/release to the configuration file
  • name attentionReactionSystemGUI
    • specifies the name of the module (used to form the stem of module port names)
